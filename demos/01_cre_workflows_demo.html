01
Search Automation
Automated Property Search & Client Alert Engine
Every client gets a profile: name, email, phone, search params, frequency. Each morning the engine queries CoStar, Crexi & LoopNet, compares against baseline, and fires personalized emails and texts — only when something new hits.
1Enter client name + search params (sq ft, budget, type, map zone)
2Engine runs daily against all 3 platforms
3Diff against yesterday's results — new listings only
4Branded email + SMS: "2 new spaces. Here's the link. Book a call."
CoStar API Crexi API LoopNet API Gmail / G Suite SMS
02
Report Generation
Smart Branded Property Report Builder
Replace the manual CoStar PDF grind. Feed search results into a branded report engine: photos, square footage breakdowns, asking rate, build year, traffic count, map link, cross-streets — formatted to stand out, not blend in.
1Search runs → results returned via API
2Template auto-fills: client name, search criteria, property data
3Photos pulled, Google Maps link generated per property
4One-click: download PDF or send directly to client
CoStar API Crexi API Google Maps PDF Engine
03
Tenant & Buyer Outreach
Flyer Upload → Lead Discovery → Outreach Campaign
Upload any listing flyer. AI reads the property type, size, and location. It then scours public lease data, RE Daily News, and web sources to build a targeted lead list — then fires personalized emails to everyone who might be interested.
1Upload flyer → AI extracts property profile
2Crawl lease databases for recent tenants in matching sq ft range
3Build lead list: name, email, company, last lease details
4Send personalized campaign — "I saw you leased on X…"
RE Daily News RE² AI Web Scrape Gmail AI Drafting
04
Owner Intelligence
Property Owner Finder & Skip Trace Engine
CoStar hits a dead end — no owner info. This workflow chains together Pima County Maps (GIS parcel data), Arizona Corporation Commission, LinkedIn, and public records to drill down to the actual human behind any LLC, find their contact info, and build a relationship-ready dossier so you can make a move before anyone else.
1Address from CoStar → Pima Maps → parcel number + LLC name
2AZ Corp Commission search → statutory agent, principals, mailing address
3LinkedIn / Facebook / web lookup for each principal
4Output: contact card with email, phone, social profiles, property history
Pima Maps GIS AZ Corp Commission LinkedIn ATTOM Data Web Scrape CoStar
05
Off-Market Prospecting
Unlisted Property Hunter & Cold Outreach
Got a buyer with specific needs — 2,000 sq ft on an acre — but nothing's listed? The engine finds every matching property in the metro (listed or not), traces the owner, and sends a targeted pitch: "I have a buyer who'd love your building."
1Input buyer needs → system searches all parcels + records
2Match unlisted properties by size, zoning, type
3Run WF 04 skip trace on each match
4Blast personalized cold outreach — "I have a buyer…"
Tax Parcel DB Pima Maps CoStar Gmail

THE TECH
BEHIND IT

Each workflow maps to a set of data sources, APIs, and AI tools. This is the technical breakdown — what's needed to build, what it connects to, and what it outputs. Phase 1 targets Workflows 01 & 03 first.
01
Search & Alert Engine
Scheduled job (4am daily) pulls listings from CoStar, Crexi, LoopNet APIs. Diffs against stored baseline per client profile. Fires branded email via G Suite and optional SMS. CRM auto-updates with last-sent timestamp and property IDs.
Outputs: Daily email digest · SMS alert · CRM log entry · Property diff report
Stack
CoStar API Crexi API LoopNet API Node / Python G Suite SMTP Twilio SMS SQLite / Postgres Cron Scheduler
02
Branded Report Builder
Takes API results and populates a custom HTML/PDF template. Pulls property photos, generates Google Maps embed links per address, adds cross-street and traffic count data, explains sq ft breakdowns in plain English. Outputs a PDF that looks nothing like CoStar's boilerplate.
Outputs: Branded PDF report · Google Maps links · Plain-English property summaries
Stack
CoStar/Crexi APIs Google Maps API Puppeteer / WeasyPrint HTML Template Engine Claude AI (summaries) Dropbox / Drive
03
Flyer → Lead → Campaign
PDF/image flyer uploaded → Claude extracts property type, size, location, target tenant profile. Crawler hits RE Daily News, Crexi public lease records, and web sources. Builds CSV lead list. Claude drafts personalized emails per lead. G Suite sends the campaign. Responses categorized: interested / not interested / follow up.
Outputs: Lead list CSV · Personalized email drafts · Campaign dashboard · Response tracker
Stack
Claude AI (vision + drafting) RE Daily News scraper RE² AI Playwright crawler G Suite API Campaign tracker DB
04
Owner Skip Trace Engine
Address → Pima County GIS API → parcel number, legal entity name. AZ Corp Commission search → principals, statutory agent, mailing address. LinkedIn / web lookup on each principal. ATTOM data cross-reference. Final output: contact card ready for outreach or CRM import.
Outputs: Owner contact card · LLC chain map · LinkedIn profile · CRM-ready JSON
Stack
Pima County GIS AZ Corp Commission ATTOM Data API LinkedIn scraper Claude AI (entity matching) Graph DB (relationships)
05
Off-Market Property Hunter
Buyer profile entered (sq ft, type, budget, area). System queries tax parcel database, Pima Maps, and county records for matching properties — regardless of listing status. For each match, runs WF 04 skip trace. Outputs a hit list of potential sellers + auto-drafted cold outreach emails per owner.
Outputs: Off-market property list · Owner contacts · Cold email drafts · Deal potential score
Stack
Tax Parcel DB Pima Maps GIS WF 04 (skip trace) Claude AI (scoring + drafting) G Suite API Zoning data
